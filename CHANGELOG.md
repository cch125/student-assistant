# 更新日志

本项目采用语义化版本号。每次大更新都会保留 Git 提交历史、创建版本标签并推送到 GitHub，因此可以随时查看或恢复之前的版本。

## [0.15.0] - 2026-07-20

### 新增

- 新增 Windows 团队知识库一键恢复脚本，组员从 GitHub 下载项目后可将知识库快照恢复到自己的本地 RAGFlow。
- 默认恢复全部 5 个知识库和 692 条知识库记录；也可只恢复核心服务卡片与第一阶段综合库。
- 自动重建 A/B/C 的 500、800、1200 tokens 分块流水线，分批上传文件并触发解析。
- 新增恢复指南，连接与导入页面增加零预算本地恢复入口。

### 安全与可靠性

- 导入前校验完整知识库快照的 SHA-256，同名文档自动跳过，可安全重复执行。
- RAGFlow API Key 仅保存在当前 PowerShell 进程，脚本结束后立即清除，不写入 GitHub或恢复报告。
- 保留仅供下载的原始附件，不对原快照中没有分块的文档强制解析。
- 明确标注 Vercel 无法访问组员本机 localhost，避免把本地恢复误认为线上共享 RAGFlow。

## [0.14.0] - 2026-07-20

### 新增

- 新增 Next.js 16 全栈 Web 版本，可由 v0 导入并直接部署到 Vercel，包含学生助手、数据看板和连接与导入三个页面。
- 每位组员可在浏览器中配置自己的 RAGFlow HTTPS 地址、API Key、问答知识库和通知补充知识库。
- 新增 Vercel Server API 代理，支持连接验证、知识库读取、文档状态、文件上传、解析触发和可信检索。
- 支持将 GitHub 中 5 套知识库快照分批导入组员选择的 RAGFlow 知识库，同名文件自动跳过。
- Web 助手支持文字与照片输入；照片先由服务器端视觉模型提取检索线索，再查询 RAGFlow。

### 安全与部署

- API Key 默认只保存在浏览器会话中，不写入服务器、日志或 GitHub；用户可主动选择仅在当前浏览器记住。
- 公网代理强制 RAGFlow 使用 HTTPS，并拒绝 localhost、回环地址和内网地址，降低 SSRF 与密钥明文传输风险。
- 查询继续使用相似度阈值和知识库原文摘录；无可靠检索结果时明确拒答。
- 使用当前修复版本 Next.js 16.2.10，并通过 TypeScript、生产构建、Python 单元测试和桌面/移动端页面检查。

## [0.12.0] - 2026-07-19

### 新增

- 新增 RAGFlow 原生图片同步器，将 MinerU 清洗后的图片和表格截图以 `image_base64` 写入 RAGFlow，由 RAGFlow 对象存储生成真实 `image_id`。
- A/B/C 三套实验知识库各写入并验证 26 个原生图片块，共 78 个；读取接口全部返回有效图片。
- 学生助手新增 JPG、PNG、WebP 照片输入，可识别表格、截图和通知照片，再使用识别结果检索知识库。

- 上传照片只在内存中缩放并发送给视觉模型，不写入本地磁盘；界面提醒学生先遮挡个人信息。
- 视觉模型只提取脱敏后的检索线索，图片内的指令按不可信文字处理；最终回答继续经过 RAGFlow 检索、来源校验和拒答阈值。
- 每日自动更新在多模态文档解析后自动执行原生图片同步；同步过程幂等，已有且有效的 `image_id` 不会重复创建。

### 验证

- 使用真实转专业表格完成照片提问端到端测试，正确命中 `转专业申请表.md`，相似度为 `0.99`。
- 完成桌面端与移动端截图检查、Python 编译检查、RAGFlow 图片读取检查和重复同步检查。

## [0.10.0] - 2026-07-19

### 新增

- 建立正式核心召回实验：40 个独立意图扩展为 160 种问法，并按意图隔离训练集与验证集。
- 加入 Rerank 对照实验，正式助手启用 `BAAI/bge-reranker-v2-m3`，验证集 Recall@1、MRR 与无答案拒答率均为 100%。
- 扩展通知类测试集，覆盖竞赛通知、未来结果、私人分数、虚构比赛和天气等边界问题。
- 助手在核心服务卡片未命中时，可严格阈值检索综合知识库并返回带暨南大学官方来源的原文摘要。
- 加入账号密钥、个人隐私、医疗诊断和录取预测四类独立安全规则，不依赖相似度阈值做风险判断。
- 新增多模态质量门禁、链接有效性检查、过期资料清单和每周自动增量更新任务。
- 新增 Docker 部署、健康检查、访问频率限制、请求大小限制和安全响应头。

### 数据与清洗

- 官网采集改为按 15 个数据源均衡分配页数，并保留历史清单中的有效页面，避免单一站点挤占采集额度。
- 清洗阶段过滤栏目首页、部门介绍和非学生事务噪声，同时恢复遗漏但仍有效的教务处文章。
- 当前保留 129 份高价值网页、34 份核心服务卡片和 17 份 MinerU 多模态文档；A/B/C 实验库使用同一套 180 份语料。
- MinerU 后处理升级到 1.1：图片补充同页上下文说明，空表格从检索单元移除；17 份文档质量检查零警告。
- 检查 166 个官网或附件链接，当前未发现失效链接；44 份较旧资料列入人工时效复核清单。
- 学生事务官方资料来源覆盖 20/20 类，核心服务卡片覆盖 17/20 类，二者在看板中分别展示。

### 实验结果

- 核心服务卡片推荐配置：向量权重 `0.90`、全文权重 `0.10`、阈值 `0.20`、Rerank 开启；160 条问法的验证集 Recall@1、MRR 与拒答率均为 100%，正负分数间隔为 `0.2680`。
- 官方通知回退推荐配置：C 库 231 个分块、向量权重 `0.90`、阈值 `0.75`；5 条通知正例与 15 条无答案问题的验证指标均为 100%，正负分数间隔为 `0.0202`。
- 通知库 Rerank 对照同样达到 100%，但未优于基础召回，因此遵循控制变量实验结论，不在通知回退中增加额外延迟。

### 运维

- 已安装 Windows 计划任务 `JNU Student Assistant Weekly Refresh`，每周日 03:00 自动采集、清洗、质检并同步 RAGFlow。
- 新增 `OPERATIONS.md`、`.env.example`、`Dockerfile` 与 `compose.yaml`，密钥只从环境变量或本机配置读取。

## [0.9.0] - 2026-07-19

### 新增

- 新增 30 个可回答问题和 10 个无答案/高风险问题组成的严格检索测试集。
- 新增 RAGFlow API 自动调优脚本，对 A/B/C 分块方案、向量权重和相似度阈值执行粗搜与精搜。
- 使用固定训练集与独立验证集评估 Recall@1/3/5、MRR、拒答率、响应时间和综合目标。
- 看板新增检索参数自动调优结果，并明确区分旧的 8 题初筛和正式严格测试。

### 实验结果

- 推荐实验库 C：142 个分块，向量相似度权重 `0.75`，最低相似度阈值 `0.70`，`top_k=10`。
- 验证集 Recall@1、Recall@3、MRR 和无答案拒答率均为 100%。
- 最低正确相关分数约 `0.7017`，最高无答案分数约 `0.6913`，安全间隔约 `0.0104`；阈值附近必须拒答或复核。
- 账号密码、个人隐私和医疗建议不依赖相似度判断，继续由独立安全规则强制拒答。
