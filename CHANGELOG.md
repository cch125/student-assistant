# 更新日志

本项目采用语义化版本号。每次大更新都会保留 Git 提交历史、创建版本标签并推送到 GitHub，因此可以随时查看或恢复之前的版本。

## [0.8.0] - 2026-07-19

### 新增

- 新增 MinerU 后的 Python 二次清洗：过滤页码、重复页眉页脚和装饰内容，并保留来源页码与坐标。
- 表格同时输出 HTML 和 JSON，图片与同页相邻文字建立来源关联，17 份附件全部完成结构化处理。
- 新增 RAGFlow A/B/C 数据流水线，分别使用 500/10%、800/10%、1200/15% 的分块与重叠设置。
- 三套实验知识库使用相同的 97 份清洗语料，共完成 291 次流水线解析并生成 291 条 RAGFlow 日志，零失败。
- 新增 8 个固定问题的检索对照测试，输出命中率、首个相关结果排名、MRR、JSON 和 CSV 报告。
- 数据看板新增三套实验知识库、文件列表、真实日志入口、分块数量和检索评测指标。

### 实验结果

- A：181 个分块，8/8 命中，MRR 0.917。
- B：148 个分块，8/8 命中，MRR 0.917。
- C：142 个分块，8/8 命中，MRR 0.938；当前固定问题集上表现略优。

### 变更

- 实验知识库改为绑定 RAGFlow 数据流水线后再上传和解析，不再使用无流水线日志的直接解析路径。

## [0.7.1] - 2026-07-19

### 修复

- MinerU 清洗成功后不再删除原始 PDF、Excel 等附件。
- 原始附件以不解析方式保留在 RAGFlow 文件列表，提供查看和下载通道。
- 检索仍使用 MinerU Markdown，避免原附件和清洗文本产生重复分块。
- 数据看板将原附件标记为“附件留存（不解析）”，不再误显示为处理中。

## [0.7.0] - 2026-07-19

### 新增

- 新增 MinerU 3.4.4 独立 Python 3.12 运行环境，模型与缓存放在 D 盘。
- 新增 PDF、图片、DOCX、PPTX、XLSX 的 OCR、版面、表格和图片多模态清洗流程。
- 新增 SHA-256 增量清洗清单，未变化文件自动跳过，失败与不支持格式明确记录。
- 新增 MinerU Markdown 自动导入 RAGFlow 第一阶段综合知识库。
- MinerU 文档成功入库后自动移除同名原始附件，避免重复检索。
- 数据流程看板新增 MinerU 成功、失败、不支持、图片、结构化 JSON 与入库状态。
- 新增 MinerU 入库检索回归测试，验证清洗文档能够被实际召回。

## [0.6.0] - 2026-07-19

### 新增

- 数据看板新增学生事务类别与责任部门覆盖报告。
- 自动区分“核心卡片”“仅有资料”和“未覆盖”三种状态。
- 自动生成按高、中、低优先级排序的下一轮数据补充清单。
- 历史未回答问题自动归并，并判断是否可能已由当前服务卡片补充。
- 新增 `/api/coverage` JSON 接口和本地 `coverage_report.json` 报告。

## [0.5.0] - 2026-07-19

### 新增

- 问答页直接显示最多 3 个达到可信阈值的相关结果，不再把候选结果只藏在匹配片段中。
- 每个相关结果独立提供官方说明和附件下载入口。
- 回答标题显示本次找到的可信结果数量。

## [0.4.1] - 2026-07-19

- 丝路奖学金旧附件返回 403 时只提供官方栏目入口，不再显示失效下载按钮。

## [0.4.0] - 2026-07-19

### 新增

- 服务卡片生成阶段自动提取暨南大学官方页面中的 PDF、Word、Excel 和压缩包附件直链。
- 助手回答优先提供“直接下载”按钮，同时保留“查看官方说明”入口。
- 通知、办事指南等没有附件的内容只显示官方说明，不会误标为可下载文档。

### 修正

- 提高标题和材料关键词精确命中的优先级，避免下载问题返回语义相近但错误的文档。
- 对需要统一身份认证、在线生成或已失效的附件明确提示，不提供不可用下载按钮。
